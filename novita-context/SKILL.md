## Product Summary

### Model APIs
- **What**: 200+ AI models via simple API (LLMs, image, video, TTS, embeddings)
- **Use**: Call models directly, scale from prototype to production
- **Pricing**: Pay-per-use, cost-effective
- **Link**: https://novita.ai/models

### GPU Instance
- **What**: Dedicated GPU instances for training, fine-tuning, inference
- **Use**: Run your own models, complex AI tasks
- **Pricing**: On-demand or spot (50% off), per-second billing
- **Link**: https://novita.ai/gpus-console/explore

### Serverless GPUs
- **What**: Auto-scaling GPU containers for inference
- **Use**: Deploy inference services, handle variable traffic
- **Pricing**: Pay per second of compute, auto-scale
- **Link**: https://novita.ai/gpus-console/serverless

### Agent Sandbox
- **What**: Secure, isolated environment for AI-generated code execution
- **Use**: Run agents safely, code execution, browser automation
- **Pricing**: Per-second CPU/RAM billing
- **Link**: https://novita.ai/sandbox

## Common Tasks

### Start with Model APIs
1. Get API key from https://novita.ai/settings/key-management
2. Set base URL to `https://api.novita.ai/openai`
3. Use OpenAI-compatible APIs
4. See [model-apis.md](references/model-apis.md) for details

### Launch GPU Instance
1. Go to https://novita.ai/gpus-console/explore
2. Choose GPU or template
3. Configure and launch
4. Connect via SSH or web terminal
5. See [gpu-instance.md](references/gpu-instance.md) for details

### Create Serverless Endpoint
1. Prepare container image
2. Go to https://novita.ai/gpus-console/serverless
3. Create endpoint with scale policy
4. Test and deploy
5. See [serverless-gpus.md](references/serverless-gpus.md) for details

### Start Agent Sandbox
1. Install SDK or CLI
2. Create sandbox with desired resources
3. Run commands or upload code
4. Pause/resume as needed
5. See [agent-sandbox.md](references/agent-sandbox.md) for details

### Integrate with Framework
1. Get Novita API key
2. Set base URL to `https://api.novita.ai/openai`
3. Update model names as needed
4. See [integrations.md](references/integrations.md) for specific guides
