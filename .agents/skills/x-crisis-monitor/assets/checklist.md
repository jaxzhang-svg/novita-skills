# Phase 1: Pre-Crisis Preparation Checklist

## Keyword Setup
- [ ] Compile complete brand keyword list (English + Chinese)
- [ ] Add all product names and variants
- [ ] Include CEO/founder/executive names
- [ ] Add slogan and tagline variations
- [ ] Compile misspellings and common typos
- [ ] Include competitor names (direct + indirect)
- [ ] Add industry-specific technical terms
- [ ] List negative sentiment keywords (high/medium/low priority)
- [ ] Include emojis and visual indicators
- [ ] Add relevant hashtags (brand + industry + negative)

## Monitoring Tools Setup
- [ ] Create X Pro monitoring columns for each keyword category
- [ ] Set up real-time streams for core keywords
- [ ] Configure Brandwatch/Meltwater AI sentiment alerts
- [ ] Set Google Alerts for brand and executive names
- [ ] Configure Industry forum monitoring (Hacker News, Product Hunt)
- [ ] Test alert delivery (ensure notifications work)
- [ ] Document false-positive patterns for filtering

## Internal Preparation
- [ ] Create crisis team contact list (PR, legal, executives, engineering)
- [ ] Establish internal communication channels (Slack, WhatsApp group)
- [ ] Document escalation procedures (who to notify for each level)
- [ ] Set up war room meeting schedule templates
- [ ] Pre-approve Level 1 response templates
- [ ] Draft Level 2-3 response templates for legal review
- [ ] Create executive statement templates
- [ ] Establish customer support escalation process

## Documentation & Playbooks
- [ ] Create crisis level classification guide
- [ ] Document response time expectations (2h acknowledgment, 4h full response)
- [ ] Create post-mortem template
- [ ] Document compensation/credit policies
- [ ] Create competitor monitoring playbook
- [ ] Establish Community Notes collaboration process
- [ ] Document media spokesperson assignments

## Training & Rehearsal
- [ ] Conduct crisis simulation drill (quarterly)
- [ ] Train customer support on de-escalation techniques
- [ ] Brief executives on social media expectations
- [ ] Train engineering team on crisis communication
- [ ] Review case studies with crisis team
- [ ] Practice drafting responses under time pressure

# Phase 2: Crisis Triage Checklist

## Initial Assessment (0-30 minutes)
- [ ] Identify the post or issue
- [ ] Capture screenshots for documentation
- [ ] Check post timestamp and age
- [ ] Note the user's account (follower count, verified status)
- [ ] Review post content and claims

## Engagement Analysis
- [ ] Count likes, retweets, comments, quote tweets
- [ ] Check if post is trending or in algorithmic feeds
- [ ] Identify key amplifiers (influencers, media accounts)
- [ ] Check cross-platform spread (Reddit, LinkedIn, forums)
- [ ] Assess engagement velocity (rate of growth)

## Content Verification
- [ ] Verify factual accuracy of claims
- [ ] Check if issue is known (support tickets, internal reports)
- [ ] Determine if content violates terms/legal policies
- [ ] Identify if it's a coordinated attack vs. individual complaint
- [ ] Assess sentiment (frustration, anger, malicious intent)

## Crisis Classification
- [ ] Determine crisis level (1/2/3) based on:
  - Engagement metrics
  - Influencer/media involvement
  - Factual accuracy
  - Brand risk assessment
- [ ] Assign urgency level
- [ ] Determine if legal review required

## Stakeholder Notification
- [ ] Notify immediate crisis team (based on level)
- [ ] Alert customer support (prepare for increased volume)
- [ ] Notify engineering if technical issue
- [ ] Alert executive team if Level 3
- [ ] Begin legal review process if Level 2-3

# Phase 3: Crisis Response Checklist

## Level 1 Response (Customer Support)
- [ ] Draft personalized response (use template, customize tone)
- [ ] Verify facts in response
- [ ] Respond publicly within 1 hour
- [ ] Invite user to DM for resolution
- [ ] Document issue in support system
- [ ] Assign to appropriate team for resolution
- [ ] Follow up with resolution update
- [ ] Monitor for escalation

## Level 2 Response (Operational)
- [ ] Draft official statement (using approved template)
- [ ] Include specific details and timeline
- [ ] Get legal review
- [ ] Get executive approval
- [ ] Respond publicly within 2 hours (acknowledgment)
- [ ] Provide full response within 4 hours
- [ ] Contact key influencers directly
- [ ] Prepare follow-up communications
- [ ] Update status page if applicable
- [ ] Monitor sentiment changes
- [ ] Prepare for potential Level 3 escalation

## Level 3 Response (Crisis Mode)
- [ ] Activate full crisis team
- [ ] Alert executive leadership immediately
- [ ] Coordinate PR, legal, executive teams
- [ ] Issue initial acknowledgment within 2 hours
- [ ] Designate official spokesperson
- [ ] Prepare holding statements for media
- [ ] Create internal communication plan
- [ ] Schedule regular updates (every 1-2 hours)
- [ ] Monitor cross-platform spread continuously
- [ ] Consider Community Notes for fact-checking
- [ ] Coordinate customer response (support team)
- [ ] Prepare post-crisis communication plan
- [ ] Document all actions for post-mortem

## Response Quality Checks
- [ ] Response is human-to-human tone (not robotic)
- [ ] Response acknowledges specific user concerns
- [ ] Response provides clear next steps or timeline
- [ ] Response is transparent about what's known/unknown
- [ ] Response matches brand voice
- [ ] Response is empathetic (not defensive)
- [ ] Response avoids blame-shifting

## Communication Channels
- [ ] Post on X (main channel)
- [ ] Update status page if applicable
- [ ] Send email to affected customers (if applicable)
- [ ] Update in-app notifications (if applicable)
- [ ] Prepare press release for media (if Level 3)
- [ ] Coordinate blog post for detailed explanation

# Phase 4: Post-Crisis Recovery Checklist

## Immediate Recovery (24-48 hours)
- [ ] Confirm full resolution of issue
- [ ] Post resolution announcement
- [ ] Monitor post-resolution sentiment
- [ ] Thank community for patience
- [ ] Address any remaining user questions
- [ ] Close open support tickets

## Data Collection & Analysis
- [ ] Capture full timeline of events
- [ ] Compile all engagement metrics
- [ ] Identify key amplifiers and influencers
- [ ] Document sentiment changes over time
- [ ] Track customer churn/cancellation metrics
- [ ] Monitor competitor mentions during crisis
- [ ] Collect user feedback on response quality

## Internal Review
- [ ] Conduct post-mortem meeting with crisis team
- [ ] Document what worked well
- [ ] Document what didn't work well
- [ ] Identify root causes
- [ ] Generate improvement action items
- [ ] Assign ownership to improvement items
- [ ] Set timelines for implementation

## Product & Process Improvements
- [ ] Compile genuine user complaints into product backlog
- [ ] Prioritize improvements based on user impact
- [ ] Share feedback with product/engineering teams
- [ ] Update playbooks based on learnings
- [ ] Refine monitoring keyword lists
- [ ] Update response templates
- [ ] Adjust escalation procedures if needed

## Relationship Recovery
- [ ] Identify most vocal critics who received resolution
- [ ] Send personalized follow-up messages
- [ ] Offer goodwill gestures (credits, gifts, extended trials)
- [ ] Convert critics to advocates where possible
- [ ] Engage community in product improvement discussions
- [ ] Showcase improvements made based on feedback

## External Communication
- [ ] Publish detailed post-mortem/blog post
- [ ] Share lessons learned transparently
- [ ] Update customers on process improvements
- [ ] Consider executive video message (if appropriate)
- [ ] Engage industry press with improvement story
- [ ] Update documentation and help center

## Documentation
- [ ] Complete post-mortem report using template
- [ ] Archive all crisis-related materials
- [ ] Update crisis management handbook
- [ ] Document all lessons learned
- [ ] Share case study with internal teams
- [ ] Update monitoring triggers and thresholds

## Long-Term Monitoring
- [ ] Track sentiment return to baseline
- [ ] Monitor for recurring issues
- [ ] Track customer retention impact
- [ ] Monitor competitor response learnings
- [ ] Review keyword list quarterly
- [ ] Update crisis team training materials
- [ ] Conduct quarterly crisis simulation

## Compensation & Remediation
- [ ] Process SLA credits for affected customers
- [ ] Issue refunds if applicable
- [ ] Send apology packages or gifts
- [ ] Extend free trials or subscriptions
- [ ] Provide feature upgrades or credits
- [ ] Document all compensation decisions
